======================================================
  인스타그램 메뉴 알림 봇 운영 가이드
  (instagram-monitor / my chatbot)
======================================================

------------------------------------------------------
1. 개요
------------------------------------------------------
인스타그램 계정(@family_koreanfood)에 새 메뉴 게시물이
올라오면 지정된 Slack 사용자들에게 자동으로 DM을 보내는 시스템.

- 플랫폼 : GitHub Actions + DynamoDB (AWS)
- 알림 채널 : Slack (my chatbot 앱 DM)
- 실행 주기 : 매일 KST 10:30 ~ 11:30, 5분 간격
- 리포지토리 : https://github.com/datalsk/instagram-monitor


------------------------------------------------------
2. 동작 흐름
------------------------------------------------------
1) GitHub Actions 스케줄러가 5분마다 워크플로우 실행
2) Playwright(헤드리스 브라우저)로 Instagram 최신 게시물 수집
3) DynamoDB에 저장된 마지막 게시물과 비교
   - 동일 → 종료 (중복 발송 없음)
   - 새 게시물 → 다음 단계 진행
4) OpenAI(gpt-4o-mini)로 메뉴 내용 요약
5) SLACK_USER_IDS에 등록된 모든 사용자에게 DM 전송
6) DynamoDB 마커 업데이트 (다음 실행 시 중복 방지)


------------------------------------------------------
3. 수신자 추가 / 제거
------------------------------------------------------
[수신자 추가 방법]
1) 추가할 팀원의 Slack User ID 확인
   - Slack에서 팀원 프로필 클릭
   - 점 세 개 메뉴 → '멤버 ID 복사' (U로 시작하는 값)

2) .env 파일 수정
   SLACK_USER_IDS=기존ID,새로운ID

3) GitHub Secret 업데이트
   - https://github.com/datalsk/instagram-monitor/settings/secrets/actions
   - SLACK_USER_IDS 항목 클릭 → 새 값으로 수정

[수신자 제거 방법]
1) .env 파일에서 해당 ID 삭제
2) GitHub Secret SLACK_USER_IDS 동일하게 수정

[현재 수신자 목록]
- U08HHKNBTMG : 이승규_프로 (Victor)
- U06BKE2ERD1 : 구재현_프로 (Tino)
- U08J9QC2UKT : (추가됨)

※ my chatbot을 Slack에 추가하지 않아도 메시지 수신 가능
※ SLACK_USER_IDS에 없는 사용자는 my chatbot 추가해도 수신 불가


------------------------------------------------------
4. 코드 수정 및 배포 방법
------------------------------------------------------
코드 수정 후 GitHub에 push하면 자동 반영됨.

  git add .
  git commit -m "수정 내용"
  git push

※ 환경 변수(.env) 값 변경 시 GitHub Secret도 함께 수정 필요
   → https://github.com/datalsk/instagram-monitor/settings/secrets/actions


------------------------------------------------------
5. 테스트 방법
------------------------------------------------------
[로컬 직접 실행]
  python run.py

[GitHub Actions 수동 실행]
  https://github.com/datalsk/instagram-monitor/actions
  → Instagram Monitor → Run workflow 버튼 클릭

[중복 방지 마커 초기화 (재발송 테스트 시)]
  AWS 콘솔 → DynamoDB → instagram-monitor-state
  → pk = 'state#family_koreanfood' 항목 삭제


------------------------------------------------------
6. 주요 환경 변수 (.env / GitHub Secrets)
------------------------------------------------------
AWS_ACCESS_KEY_ID      : AWS 액세스 키
AWS_SECRET_ACCESS_KEY  : AWS 시크릿 키
AWS_REGION             : ap-northeast-2 (서울)
OPENAI_API_KEY         : OpenAI API 키
OPENAI_MODEL           : gpt-4o-mini
SLACK_BOT_TOKEN        : Slack Bot Token (xoxb-로 시작)
SLACK_USER_IDS         : 수신자 User ID (콤마로 구분)
INSTAGRAM_ACCOUNTS     : 모니터링 계정 (@ 제외)
DYNAMODB_TABLE         : instagram-monitor-state


------------------------------------------------------
7. 인프라 리소스
------------------------------------------------------
[GitHub]
- 리포지토리    : https://github.com/datalsk/instagram-monitor
- 워크플로우    : .github/workflows/monitor.yml
- Secrets      : 리포 Settings → Secrets and variables → Actions

[AWS]
- DynamoDB 테이블 : instagram-monitor-state (ap-northeast-2)
- IAM 역할       : instagram-monitor-role
- Lambda 함수    : instagram-monitor (비활성화 상태, 미사용)


------------------------------------------------------
8. 문제 발생 시 확인 순서
------------------------------------------------------
1) GitHub Actions 실행 로그 확인
   https://github.com/datalsk/instagram-monitor/actions
   → 최근 실행 클릭 → monitor 잡 → 로그 확인

2) 주요 에러 유형
   - HTTP 429  : Instagram 요청 차단 (일시적, 재시도 대기)
   - missing_scope : Slack Bot Token 권한 부족
                    → api.slack.com/apps → OAuth & Permissions
                      → chat:write 권한 추가 후 재설치
   - 메시지 미발송 : DynamoDB 마커 확인
                    (이미 동일 게시물로 처리됐을 수 있음)
