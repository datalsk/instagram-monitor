======================================================
  인스타그램 메뉴 알림 봇 운영 가이드
  (instagram-monitor / my chatbot)
======================================================

------------------------------------------------------
1. 개요
------------------------------------------------------
인스타그램 계정(@family_koreanfood)에 새 메뉴 게시물이
올라오면 지정된 Slack 사용자들에게 자동으로 DM을 보내는 시스템.

- 플랫폼 : AWS Lambda + EventBridge + DynamoDB
- 알림 채널 : Slack (my chatbot 앱 DM)
- 실행 주기 : 매일 KST 10:30 ~ 11:30, 5분 간격


------------------------------------------------------
2. 동작 흐름
------------------------------------------------------
1) EventBridge가 5분마다 Lambda 함수 실행
2) 현재 시각이 KST 10:30~11:30 범위 밖이면 즉시 종료
3) RSS 피드에서 최신 게시물 가져오기
4) DynamoDB에 저장된 마지막 게시물과 비교
   - 동일 → 종료 (중복 발송 없음)
   - 새 게시물 → 다음 단계 진행
5) Claude AI로 메뉴 내용 요약
6) SLACK_USER_IDS에 등록된 모든 사용자에게 DM 전송
7) DynamoDB 마커 업데이트 (다음 실행 시 중복 방지)


------------------------------------------------------
3. 수신자 추가 / 제거
------------------------------------------------------
[수신자 추가 방법]
1) 추가할 팀원의 Slack User ID 확인
   - Slack에서 팀원 프로필 클릭
   - 점 세 개 메뉴 → '멤버 ID 복사' (U로 시작하는 값)

2) .env 파일 수정
   SLACK_USER_IDS=기존ID,새로운ID

3) 재배포
   python deploy_boto3.py

[수신자 제거 방법]
1) .env 파일에서 해당 ID 삭제
2) 재배포 : python deploy_boto3.py

[현재 수신자 목록]
- U08HHKNBTMG : 이승규_프로 (Victor)
- U06BKE2ERD1 : 구재현_프로 (Tino)
- U08J9QC2UKT : (추가됨)

※ my chatbot을 Slack에 추가하지 않아도 메시지 수신 가능
※ SLACK_USER_IDS에 없는 사용자는 my chatbot 추가해도 수신 불가


------------------------------------------------------
4. 재배포 방법
------------------------------------------------------
.env 파일 수정 후 아래 명령 실행:

  python deploy_boto3.py

재배포 항목:
- Lambda 함수 코드 및 환경 변수
- EventBridge 스케줄 규칙

※ DynamoDB 테이블과 IAM 역할은 이미 존재하면 건너뜀


------------------------------------------------------
5. 테스트 방법
------------------------------------------------------
1) DynamoDB 마커 초기화 (이전 게시물 기록 삭제)
   - AWS 콘솔 → DynamoDB → instagram-monitor-state
   - pk = 'state#family_koreanfood' 항목 삭제

2) Lambda 수동 실행
   - AWS 콘솔 → Lambda → instagram-monitor → 테스트

※ 시간 필터(10:30~11:30) 적용 중이므로
   해당 시간 외 테스트 시 시간 필터를 임시로 주석 처리 후 재배포 필요

   [lambda_function.py 수정]
   # if not _in_active_window():
   #     return {"statusCode": 200, "body": "outside active window"}

   테스트 완료 후 반드시 원복 및 재배포


------------------------------------------------------
6. 주요 환경 변수 (.env)
------------------------------------------------------
AWS_ACCESS_KEY_ID      : AWS 액세스 키
AWS_SECRET_ACCESS_KEY  : AWS 시크릿 키
AWS_REGION             : ap-northeast-2 (서울)
ANTHROPIC_API_KEY      : Claude AI API 키
SLACK_BOT_TOKEN        : Slack Bot Token (xoxb-로 시작)
SLACK_USER_IDS         : 수신자 User ID (콤마로 구분)
INSTAGRAM_ACCOUNTS     : 모니터링 계정 (@ 제외)
RSS_FEED_URL           : rss.app RSS 피드 URL
DYNAMODB_TABLE         : instagram-monitor-state
CLAUDE_MODEL           : claude-haiku-4-5-20251001


------------------------------------------------------
7. AWS 리소스
------------------------------------------------------
- Lambda 함수   : instagram-monitor (ap-northeast-2)
- DynamoDB 테이블 : instagram-monitor-state
- EventBridge 규칙 : instagram-monitor-schedule (rate 5분)
- IAM 역할      : instagram-monitor-role
- CloudWatch 로그 : /aws/lambda/instagram-monitor


------------------------------------------------------
8. 문제 발생 시 확인 순서
------------------------------------------------------
1) AWS CloudWatch → 로그 그룹 /aws/lambda/instagram-monitor
   → 최근 로그 스트림에서 ERROR 메시지 확인

2) 주요 에러 유형
   - missing_scope  : Slack Bot Token 권한 부족
                      → api.slack.com/apps → OAuth & Permissions
                        → chat:write 권한 추가 후 재설치
   - RSS_FEED_URL 없음 : .env에 RSS_FEED_URL 누락
   - 메시지 미발송  : DynamoDB 마커 확인 (이미 동일 게시물로 처리됐을 수 있음)
